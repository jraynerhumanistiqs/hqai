Most useful What we want from this group

You're not testing for bugs - you're testing for product instinct. Five lenses we lean on hardest.

1

Would a real SME owner click this?

Flag where the language gets too HR-jargon, or where it talks down.

2

Where does it save 80%?

vs your current consulting workflow. Be ruthless on where it saves nothing.

3

What's the missing feature?

The one thing that would make you confident handing this to a client tomorrow.

4

Where do you mistrust the AI?

Specific examples. "This clause read wrong because X" beats "AI feels off".

5

What would clients break?

Walk as your worst-case-scenario client. Tell us where they'd get stuck.

Known Limits (so you don't flag them as bugs)

  • Dark mode forced off until a full legacy-panel audit.
  • Documents library + Shortlist top header on old layout - function works.
  • DOCX / PPTX / shareable-link export removed by design - PDF only.
  • Voyage embeddings + Cohere rerank may be env-gated in pilot.
  • $25 anonymous Letter fulfilment route incomplete - don't complete in prod.
  • Resend sender / reply-to needs final-domain verification.
  • PDF / PPTX visual QA on Vercel serverless still pending.
  • Citations panel removed from generated documents on purpose.

Reporting How to flag issues

  1. Tag by page name (eg "AI Administrator", "Shortlist / phone screen", "CV Scoring").
  2. Severity in plain language (see below).
  3. Repro - one line of steps + screenshot or Loom.
  4. Suggestion if you have one - optional, no design pressure.
  5. Daily summary to Jimmy end-of-day. One Loom beats 100 screenshots.

Blocker

Stops the flow

Can't sign in. Can't generate a doc. Error screen with no recovery.

Annoying

Slows you down

Hover state wrong, dropdown closes too fast, copy reads weirdly.

Polish

Looks-and-feels

Spacing, alignment, colour, font weight. Nice-to-have not need-to-have.
