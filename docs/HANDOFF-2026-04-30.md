## Decisions Made and Why

| Decision | Rationale |
|----------|-----------|
| Structured output via forced `tool_use` for all Campaign Coach steps | Free-form JSON in SSE was unreliable; tool-use guarantees shape and lets us stream tokens while still parsing safely. |
| Service-role admin client (`supabaseAdmin`) for prescreen tables | RLS was blocking browser-side reads we needed; gated server endpoints are the cleaner workaround than loosening policies. |
| Section-aware chunker for Fair Work Act | Naive splitter produced 737 fat chunks with poor retrieval; section-aware produces 1471 sharp chunks tied to s.### references. |
| Hybrid retrieval (vector cosine OR `websearch_to_tsquery`) | Pure vector missed exact statute references; pure BM25 missed paraphrased queries. OR clause covers both. |
| Single canonical URL `hqai.vercel.app` | User's call (Option B). Avoids dual-domain SEO + cookie pain. |
| Drop `$` end-anchor from citations regex | Sonnet appends "General information, not legal advice" disclaimer AFTER the fenced ```citations``` block, which made the strict end-anchored regex miss it; raw fence then leaked into chat bubble. Fix in `lib/parse-citations.ts`. |
| Catch block sends BOTH `error` AND `done:true` events | Wizard frontend was showing "server returned: null" when stream errored before the done event. Now both fire so wizard can surface the real exception inline (no dev-tools needed). |
| Plain hyphens only in UI copy (no em/en-dashes) | User rule, saved to global memory. Code comments exempt. |
| No permission prompts mid-task | User rule (saved): assume "Just this once" approval, only pause for genuine multi-option decisions. |
